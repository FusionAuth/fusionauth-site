The [FusionAuth CLI](https://github.com/FusionAuth/fusionauth-node-cli) is a great tool for doing deeper work within FusionAuth. Things like dealing with your themes, working with email templates, or dealing with our powerful lambda functions help you get further with FusionAuth.

What if the CLI could also help you get started with FusionAuth? The latest release of the CLI does just that.

{/* more */}

## Install the CLI (or upgrade)

```
npm install @fusionauth/cli
```

<Aside nodark="true" type="caution" title="Docker Required">
Using the CLI to kickstart a FusionAuth instance requires a local version of Docker in your command-line PATH.
</Aside>


## Kickstart your educational process

We want to make sure that every new user of FusionAuth can get up and running as quickly as possible. Testing FusionAuth should be more about integrating with your application than it is about configuring your authentication provider.

In order to get you there faster, we now have a new suite of commands to get you a simple, well-configured starting point for your projects.

By running the following command, you’ll get a FusionAuth Docker compose file and a Kickstart configuration file that contains all you need to be ready to implement authentication: a FusionAuth instance, an API key, and an application. All with one command instead of tedious configuration and setup.

```
npx fusionauth kickstart:install
```

![Output of the kickstart:install command showing the setting of username, password, and application name, and how to access the newly installed FusionAuth](/img/blogs/cli-kickstart/cli-output.png)

Once your FusionAuth configuration is ready, you can move into the created directory and run commands to deal with the Docker container.

* `npx fusionauth kickstart:start`: starts your FusionAuth container and configures your instance
* `npx fusionauth kickstart:stop`: non-destructively pauses your FusionAuth container 
* `npx fusionauth kickstart:kill`: shuts down and removes your FusionAuth container

## The future of the CLI

Developers should be in whatever environment works best for them. For the most part, that’s head-down writing code in their IDE. What better way to keep you focused and productive than to keep you from having to load up the FusionAuth dashboard just to configure your application.

As we move forward with new features for the CLI, be sure to let us know what would keep you moving forward with your applications by putting any requests in the issues for [the CLI on GitHub](https://github.com/FusionAuth/fusionauth-node-cli).
