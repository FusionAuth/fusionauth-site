Say hello to FusionAuth `1.68.0`! In honor of our significant improvements to FusionAuth's MFA challenge flow, we've decided to name this one the **Intelligent Kamfa**:

![an image of a red kamfa fish with the caption 'intelligent kamfa presented an mfa challenge', image credit to redditor u/TorontoFishGuy](/img/blogs/release-1-68/kamfa.jpg "the intelligent kamfa")

This version includes intelligent MFA, a new variable for email and message templates, hashing for account recovery codes, and several new SDK versions.

![intelligent kamfa](/img/blogs/release-1-68/pokekamfa.jpg "intelligent kamfa determines exactly when to present an MFA challenge")

## Intelligent MFA

**Intelligent MFA** uses a collection of signals to trigger MFA challenges for users based on the risk level of a specific login attempt.

## BaseURL Variable

Email and message templates can now build links using the new `baseUrl` variable, instead of hard-coding a URL.

## Hashed Recovery Codes

FusionAuth now hashes recovery codes at rest in the application.

## TOTP Authenticator Names

Users can now add a custom name for each TOTP Authenticator associated with their account. This makes it easier to identify and remove outdated TOTP methods.

## SDKs

This release adds the following new SDK versions:

* [Swift SDK for iOS](/docs/sdks/swift-sdk) `1.3.0` with support for iOS `26.5.x`.
* [Kotlin SDK for Android](/docs/sdks/android-sdk)  `1.2.0` with support for Android API Level 36 (API Level 37 is in beta).
* [React SDK](/docs/sdks/react-sdk) `2.6.0` with support for React `19.2.x`.
* [Angular SDK](/docs/sdks/angular-sdk) `2.0.0` with support for Angular `22.0.x`.
* [Vue SDK](/docs/sdks/vue-sdk) `1.3.0` with support for Vue `3.5.x`.
