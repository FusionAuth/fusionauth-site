## Overview

Add login, registration, multifactor authentication (MFA), single sign-on (SSO), and much more to your apps in days, not months, using FusionAuth.

In this quickstart you:
- Create a basic app.
- Add a public page and a protected account page.
- Add registration, login, and logout.
- Add refreshing of the authentication cookie to the protected page.

## Before you start

To use this quickstart you need:

- [Docker](https://www.docker.com) version 23 or later to run the container with FusionAuth.
- [Node.js](https://nodejs.org/en) version 22 or later to run the React app.
- A project directory to work in.

The <a href={frontmatter.codeRepository}>repository for this quickstart</a> includes the code for the different stages in four directories:

- `01_start`: Empty files.
- `02_noAuthentication`: The example app with placeholder pages and no authentication.
- `03_withAuthentication`: Adds authentication that uses FusionAuth.
- `04_callTheServer`: Adds refreshing the authentication cookie to the account page.

You can start from any of these points, or jump ahead to one of them if you get stuck.

### Run the base app

**Step 6:** Open a terminal and change your current directory to the project directory.

**Step 7:** Run the commands below to install the Node.js packages for React.

```sh
npm install  @fusionauth/react-sdk@~2.5.1  react@~19.1.1  react-dom@~19.1.1  react-router-dom@6.7.0

npm install --save-dev  @eslint/js@~9.33.0  @types/react@~19.1.10  eslint@~9.33.0  typescript@~5.9.2  typescript-eslint@~8.40.0  parcel@~2.15.4
```

**Step 8:** Run the command below to start the web server.

```sh
npx parcel index.html --host 0.0.0.0 --port 3000
```

**Step 9:** Open a browser and view http://localhost:3000 to see the site.

![Login page with no authentication](../../../public/img/quickstarts/quickstart-javascript-react-web/login.webp)

## Call FusionAuth only when authenticated

Once a user has successfully logged in, FusionAuth stores an `http-only` cookie in their browser. If you use a full web server instead of this example's static file host in the future, you can check for the presence of this cookie to verify that the user is authenticated. For calls to the FusionAuth API, you need an API key, and thus for security's sake you need to make the calls from a server instead of the browser. Since this example doesn't use a server, almost no calls to the FusionAuth API can be made.

As an illustration of calling FusionAuth from the browser only when you're logged in, in this section you add a call to [refresh the user's authentication cookie](/docs/apis/hosted-backend#refresh). (Note that this call isn't necessary because you set `shouldAutoRefresh: true` in the `FusionAuthProviderConfig` in `index.tsx`, which refreshes the user's cookie when it expires anyway.)
