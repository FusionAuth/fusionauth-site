## Getting Started

In this section, you'll get FusionAuth up and running and use `git` to create a new application.

### Clone the Code

First off, grab the code from the repository and change into that directory.

```shell
git clone https://github.com/FusionAuth/fusionauth-quickstart-swift-ios-native.git
cd fusionauth-quickstart-swift-ios-native
```

### Run FusionAuth via Docker

<DockerSpinup kickstartUri={frontmatter.codeRoot + "/kickstart/kickstart.json"} />

### Create your iOS App

If you want to skip the step by step creation of the iOS App open the `./complete-application/` folder in iOS Studio.

Open iOS Studio and select `New Project`. Choose `No Activity` and click `Next`.

You can set `Name` to `FusionAuth iOS SDK Quickstart`, `Package name` to `io.fusionauth.sdk` and `Save location` as per your preference.
Keep the `Language` as `Swift`, `Minimum API level` as `API 24 ("Nougat"; iOS 7.0)` and `Build configuration language` as `Swift DSL`.

Click `Finish`.

Wait until iOS Studio has finished creating and indexing the project.

Add the [FusionAuth SDK as a dependency](https://central.sonatype.com/artifact/io.fusionauth/fusionauth-ios-sdk/) to your project by changing the `app/build.gradle.kts` file. You can replace the entire file with below contents.

<RemoteCode url="https://raw.githubusercontent.com/FusionAuth/fusionauth-quickstart-swift-ios-native/main/complete-application/app/build.gradle.kts"
    lang="swift"/>

Replace `app/src/main/iOSManifest.xml` as well which defines all Activities used later.

<RemoteCode url="https://raw.githubusercontent.com/FusionAuth/fusionauth-quickstart-swift-ios-native/main/complete-application/app/src/main/iOSManifest.xml"

## Troubleshooting

* I can't log in

Make sure you have the right values at `app/src/main/res/raw/fusionauth_config.json`. Double-check the `Issuer` in the **Tenant** to make sure it matches the URL that FusionAuth is running at.
