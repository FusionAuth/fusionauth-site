This quickstart adds login and a protected page to a React web app using a preconfigured, local version of FusionAuth in the following steps:

1. Create a new React app.
1. Customize the app with home and account pages.
1. Add login to protect the account page.
1. Add a button to the account page to fetch the user's information.

## Prerequisites

This Quickstart requires:

* [Node.js](https://nodejs.org/en/download) 22 or later
* [Docker](https://docs.docker.com/get-started/get-docker/) 23 or later
* On macOS and Windows, one of the following container management tools:
  * [OrbStack](https://docs.orbstack.dev/quick-start) (to use Orbstack for `docker compose` commands after install, run `docker context use orbstack` )
  * [Podman](https://podman.io/docs/installation) (in the commands below, replace `docker` with `podman`)
  * [Docker desktop](https://www.docker.com/products/docker-desktop/)

Be sure to open your container management tool to configure and install any dependencies needed by that tool, including Rosetta and Developer Tools on macOS.

## Install and Run FusionAuth

To begin, clone the [GitHub repository for this Quickstart](https://github.com/FusionAuth/fusionauth-quickstart-react) to your local machine:

```bash
git clone git@github.com:FusionAuth/fusionauth-quickstart-react.git
```

The repository contains:

* `fusionauth-backend`, a preconfigured version of FusionAuth
* `react-frontend-steps`, a runnable version of each development step of the React application

In a terminal, navigate to `fusionauth-quickstart-react/fusionauth-backend`. To start a local instance of FusionAuth, run the following command:

```bash
docker compose up -d
```

<Aside type="note">
During the Docker compose process, the included `/fusionauth-quickstart-react/kickstart/kickstart.json` file uses the FusionAuth API to:

* Generate an API key
* Configure CORS (<Breadcrumb>Settings -> System -> CORS</Breadcrumb>)
* Create your app ("Quick Start App") with appropriate settings and redirect URIs (<Breadcrumb>Applications -> Edit Application</Breadcrumb>)
* Create an admin user (admin@example.com) and register the user with your app (<Breadcrumb>Users</Breadcrumb>)
* Create a user (richard@example.com) and register the user with your app
* Create a custom theme ("Quick Start Theme") using `fusionauth-quickstart-react/kickstart/css/styles.css` and apply it to your default tenant (<Breadcrumb>Customizations -> Themes</Breadcrumb>)
</Aside>

Once Docker compose completes, open a browser window and navigate to the admin UI of your FusionAuth instance at [http://localhost:9011/admin](http://localhost:9011/admin). Use the following admin account credentials to log in:

* Username: `admin@example.com`
* Password: `password`

After authenticating, you should see the FusionAuth admin UI, which shows information about active users, logins, registrations, and more:

![The FusionAuth admin UI.](/img/quickstarts/react/fusionauth-admin.png)

## Create the React app

First, create a new TypeScript React app with Vite: 

1. To generate a new React app in a folder called `app-quickstart-react`, open a terminal window and run the following command:

   ```bash
   npm create vite@latest app-quickstart-react -- --template react-ts
   ```

   Select "Yes" to install necessary Vite packages if prompted. Then select "No" for the following options:

   * Use rolldown-vite (Experimental)? **No**  
   * Install with npm and start now? **No**

1. Navigate into the app folder and install the project dependencies with the following commands:

   ```bash
   cd app-quickstart-react
   ```

   ```bash
   npm install
   ```

   ```bash
   npm install react-router-dom
   ```

1. Delete the file `app-quickstart-react/src/App.tsx` as you'll add routing to display the app pages:

   ```bash
   rm -rf src/App.tsx
   ```

<Aside type="note">
This FusionAuth configuration specifies the following values:

* `redirectUri`: URI to open after a successful login.  
* `postLogoutRedirectUri`: URI to open after logout.  
* `shouldAutoRefresh`: Set to automatically renew the access token.  
* `shouldAutoFetchUserInfo`: Return data for the user as part of a successful login.  
* `scope`: OAuth 2.0 permissions for the access token.  
* `clientId`: ID of the application in the FusionAuth instance.  
* `serverUrl`: Base URL for the FusionAuth authorization server.  
* `onRedirect`: Callback that executes when a redirect occurs.
</Aside>
