<p>
JWTs are portable, stateless tokens and are often used to communicate identity between systems. Every signed JWT has three parts:
</p>
<ul>
<li>
<b>A header</b>, which contains metadata, including information about the key used to sign the JWT.
</li>
<li>
<b>A body</b>, which is a JSON object with an arbitrary payload; the keys of this JSON object are commonly called “claims”.
</li>
<li>
<b>A signature</b>, which is built by performing a cryptographic operation over the header and the body.
</li>
</ul>
<p>
These are all base64 URL encoded so the resulting string is safe to put in HTTP headers, cookies and elsewhere. Every signed JWT lets you verify the integrity of the JWT without contacting the signer.
</p>
<p>
This decoder lets you examine the contents of any JWT by pasting it into the <b>Token</b> form field. It’ll automatically decode the values and place the header and body into the respective fields.
</p>
<p>
Use this tool to confirm your JWT is being generated with the metadata and claims you expect, or to examine JWTs other systems are creating to discover their properties.
</p>
