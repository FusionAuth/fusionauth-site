## Prerequisites

* A FusionAuth instance running on a publicly accessible URL. You can spin up a [basic FusionAuth Cloud instance](/pricing) or [install it on any server](/docs/get-started/download-and-install).
* An [Okta Workforce Identity Cloud Account](https://www.okta.com) connected to a business email address.
** The [Okta Verify app](https://help.okta.com/en-us/Content/Topics/Mobile/okta-verify-overview.htm) if your Okta account is not configured to send SMS messages for 2FA.

## Okta Configuration

Log in to Okta and navigate to the <Breadcrumb>Admin</Breadcrumb> panel.

![Okta admin button.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-admin-button.png)

Navigate to <Breadcrumb>Applications -> Applications</Breadcrumb> and click the <Breadcrumb>Create App Integration</Breadcrumb> button.

![create app integration.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-create-app-integration.png)

Select <Breadcrumb>OIDC - OpenID Connect</Breadcrumb> and <Breadcrumb>Web Application</Breadcrumb> then click <Breadcrumb>Next</Breadcrumb>.

![OIDC and web application.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-oidc-webapp.png)

Enter an **App integration name** and ensure that **Authorization Code** is checked. Then, supply a **Sign-in redirect URI** of the form `<YOUR_FUSIONAUTH_URL>/oauth2/callback`, where **YOUR_FUSIONAUTH_URL** is the URL of your FusionAuth instance. For example, `https://local.fusionauth.io`.

![name and redirect URI.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-name-uri.png)

Under the <Breadcrumb>Assignments</Breadcrumb> section, select **Skip group assignment for now**. Hit <Breadcrumb>Save</Breadcrumb>.

![skip group assignment.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-skip-group.png)

Copy the **Client Id** and **Secret** into a text file for later use.

![client id and secret.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-client-id-secret.png)

Under <Breadcrumb>Sign On</Breadcrumb>, navigate to the section <Breadcrumb>OpenID Connect ID Token</Breadcrumb> and change the **Issuer** to use the **Okta URL**. Hit <Breadcrumb>Save</Breadcrumb> and copy this URL into a text file.

![issuer URL.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-issuer.png)

Navigate to <Breadcrumb>Directory -> People</Breadcrumb> and click on the username of the user that you'd like to authenticate. Then click <Breadcrumb>Assign Applications</Breadcrumb> and hit the <Breadcrumb>Assign</Breadcrumb> button next to the one you created. Then click <Breadcrumb>Save and Go Back</Breadcrumb> and <Breadcrumb>Done</Breadcrumb>.

## Configure a new FusionAuth OpenID Connect Identity Provider

Although FusionAuth doesn't include an Okta identity provider, the OpenID Connect identity provider works as Okta supports the standard OpenID Connect protocols.

Navigate to your FusionAuth instance. Select <Breadcrumb>Settings</Breadcrumb> from the sidebar and then navigate to the section <Breadcrumb>Identity Providers</Breadcrumb>.

Select <Breadcrumb>Add OpenID Connect</Breadcrumb> from the <Breadcrumb>Add</Breadcrumb> dropdown at the top right of the page.

![Create a new OpenID integration](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-choose-openid.png)

- Provide a **Name**, like `Okta`.
- Set **Client Id** to the **Client ID** recorded when creating the app integration on Okta.
- Select `HTTP Basic Authentication` for the **Client Authentication** field.
- Set the **Client secret** to the client secret recorded when creating the app integration on Okta.
- Enable **Discover endpoints**.
- Use the Okta URL that you recorded as the **Issuer** URL.
- For the **Scope**, type in `openid email`.
- Optionally, change the **Button Text** field to `Login with Okta`.

![identity provider settings.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-settings-identity-provider.png)

- Navigate to the <Breadcrumb>Applications</Breadcrumb> section and ensure that the application you are adding Okta authentication to has the <Breadcrumb>Enabled</Breadcrumb> and **Create Registration** switches toggled. Hit <Breadcrumb>Save</Breadcrumb>.

## Testing the Login

To test, navigate to the applications page in FusionAuth. Click on the <Breadcrumb>View</Breadcrumb> icon (magnifying glass) next to the application you enabled Okta login on and copy the `OAuth IdP login URL` address. Navigate to this address. You should see a <Breadcrumb>Login with Okta</Breadcrumb> button on your app's sign-in page:

![result.](/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/okta-idp-result.png)

Click the <Breadcrumb>Login with Okta</Breadcrumb> button. Test logging in with the username and password, as well as the 2FA method selected when configuring your account on Okta.

If it is all set up correctly, you should be redirected back to your app, successfully logged in. The user will be added to FusionAuth, and you can examine the <Breadcrumb>Linked accounts</Breadcrumb> section of the user details screen to see that the Okta OIDC link was created.
