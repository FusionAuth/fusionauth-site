SendGrid is a transactional email service. Using a SAML v2 identity provider, you can control who has access to your SendGrid account. This document covers the configuration necessary to get SendGrid working with FusionAuth as the identity provider using SAML v2. 

<Aside type="note">
This document helps you set up single sign-on using SAML for your SendGrid account. If you are looking to set up SendGrid as an email provider, you want [this documentation instead](/docs/customize/email-and-messages/configure-email#sendgrid-twilio).
</Aside>

## Prerequisites

This document assumes you have a running instance of FusionAuth and a working SendGrid application. You will also need an admin account for FusionAuth and be the account owner in SendGrid to configure them correctly.

<Aside type="note">
Single Sign-On (SSO) is available for Twilio SendGrid Email API Pro, Premier, and Marketing Campaigns Advanced plans only. See the [Twilio SendGrid pricing page](https://sendgrid.com/pricing/) for a full list of Twilio SendGrid features available by plan.
</Aside>

## Configure SendGrid

In your [SendGrid dashboard](https://app.sendgrid.com/), navigate to <Breadcrumb>Settings -> SSO Settings</Breadcrumb> and click <Breadcrumb>Add Configuration</Breadcrumb>.

![Adding SSO configuration in SendGrid.](/img/docs/lifecycle/authenticate-users/integrations/saml/sendgrid-sso-settings.png)

Name it `FusionAuth` and write down both **Single Sign-On URL** and **Audience URL (SP Entity ID)** fields as you'll use them soon. Click <Breadcrumb>Next</Breadcrumb>.

![Filling SSO configuration in SendGrid.](/img/docs/lifecycle/authenticate-users/integrations/saml/sendgrid-add-configuration-values.png)

To get the needed values for the below form, you need to configure FusionAuth.

![Filling SSO configuration in SendGrid.](/img/docs/lifecycle/authenticate-users/integrations/saml/sendgrid-add-configuration-after-initial.png)

## Configure FusionAuth

Before finishing configuring SendGrid, you have to configure the certificate issuer and create an application in FusionAuth.

Log in to your instance, navigate to <Breadcrumb>Tenants</Breadcrumb> and click <Icon name="edit" /> on your tenant to edit it. Change the **Issuer** to your FusionAuth instance domain without the protocol (e.g. `your.application.com`) and click on <Icon name="save" /> to save your tenant.

Now, navigate to <Breadcrumb>Applications</Breadcrumb>, <Icon name="plus" /> and name it `SendGrid`.

![Adding an Application in FusionAuth.](/img/docs/lifecycle/authenticate-users/integrations/saml/add-application.png)

Go to the <Breadcrumb>SAML</Breadcrumb> tab and toggle the <Breadcrumb>Enabled</Breadcrumb> switch. Paste the **Audience URL (SP Entity ID)** and **Single Sign-On URL** you copied from SendGrid to **Issuer** and **Authorized redirect URLs** fields in your FusionAuth application, respectively.

_Mapping fields_

| SendGrid                                               | FusionAuth                                          |
|--------------------------------------------------------|-----------------------------------------------------|
| **Audience URL (SP Entity ID)** | **Issuer**                   |
| **Single Sign-On URL**          | **Authorized redirect URLs** |

Scroll down to <Breadcrumb>Authentication response</Breadcrumb> and change **Signature canonicalization method** to `Exclusive`.

Click <Icon name="save"/> to save your application. When redirected to the <Breadcrumb>Applications</Breadcrumb> page, view your application details by clicking <Icon name="fa-search"/>.

![Getting the SAML integration details.](/img/docs/lifecycle/authenticate-users/integrations/saml/saml-integration-details.png)

In <Breadcrumb>SAML v2 Integration details</Breadcrumb>, write down both **Entity Id** and **Login URL** fields.

Copy the **Metadata URL** and open it in a new browser tab. This will contain configuration you'll need shortly.

<Aside type="note">
This URL contains an XML file. If your browser doesn’t automatically format it, you can copy everything and use [an online XML beautifier](https://xmlviewer.org/).
</Aside>

Copy the contents of the `X509Certificate` element from the XML file and write it down.

![Copying certificate from SAML metadata.](/img/docs/lifecycle/authenticate-users/integrations/saml/metadata.png)

## Finish setting up SendGrid

Retrieve the first two values you copied from your FusionAuth application and write the **Entity Id** to SendGrid's **SAML Issuer ID** and the **Login URL** in **Embed Link**.

![Filling SSO configuration in SendGrid.](/img/docs/lifecycle/authenticate-users/integrations/saml/sendgrid-add-configuration-after-initial.png)

In the <Breadcrumb>Certificates</Breadcrumb> session, click <Breadcrumb>Add Certificate</Breadcrumb>.

In the modal, paste your FusionAuth application's certificate into **X509 Certificate** and enclose it between `-----BEGIN CERTIFICATE-----` and `-----END CERTIFICATE-----`, as seen in the example below.

```
-----BEGIN CERTIFICATE-----
<contents of X509Certificate copied from FusionAuth>
-----END CERTIFICATE-----
```

Click <Breadcrumb>Add Certificate</Breadcrumb> to save the certificate.

Finally click <Breadcrumb>Enable SSO</Breadcrumb> to finish the configuration.

## User Management

Go to <Breadcrumb>Settings -> Teammates</Breadcrumb>, click the <Breadcrumb>Add Teammate</Breadcrumb> dropdown button in the top right corner of the page and select <Breadcrumb>Add SSO teammate</Breadcrumb>.

Fill in the same email address from a FusionAuth user and their first and last names. Click <Breadcrumb>Next</Breadcrumb>. Choose one of the available permissions (e.g. `Add with read-only access`) and click <Breadcrumb>Save</Breadcrumb> to add the user.

## Testing

Browse to the [SendGrid login page](https://app.sendgrid.com/) in another browser or in an incognito tab and click <Breadcrumb>Log in with single sign-on (SSO) instead</Breadcrumb>.

![SendGrid login page with SSO option.](/img/docs/lifecycle/authenticate-users/integrations/saml/login-page.png)

In the dedicated SSO page, you need to fill in the username you created earlier and click <Breadcrumb>Log In with SSO</Breadcrumb>.

![SendGrid SSO login page.](/img/docs/lifecycle/authenticate-users/integrations/saml/login-sso.png)

You'll be now redirected to the FusionAuth login screen. After entering the credentials, you arrive back at the SendGrid application already logged in.
