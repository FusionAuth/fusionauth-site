## Overview

This page explains how to use FusionAuth as an identity provider for [Tableau Cloud](https://www.tableau.com/products/cloud-bi) using OpenID Connect (OIDC).

### Prerequisites

This document assumes you have:

* Admin access to an instance of FusionAuth
* Admin access to a Tableau Cloud application

## Configure CORS In FusionAuth

Before configuring OIDC for Tableau, make sure the OIDC Tableau origin is allowed for requests. In your FusionAuth dashboard, navigate to <Breadcrumb>Settings -> System -> CORS</Breadcrumb> and toggle on the <Breadcrumb>Enabled</Breadcrumb> switch. Then add the origin, which is the URL in your browser's address bar when you are logged in to Tableau. It is something like `https://prod-ch-a.online.tableau.com` or `https://prod-usw-a.online.tableau.com`. The format is typically: `https://prod-{region}.online.tableau.com` with an `https://sso.online.tableau.com` origin.

![Configure CORS settings in FusionAuth.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/cors-configuration.png)

Under **Allowed methods**, enable both the <Breadcrumb>GET</Breadcrumb> and <Breadcrumb>POST</Breadcrumb> methods. Then save the changes.

## Create An RS256 Signing Key

Tableau Cloud requires the `RS256` algorithm for signing tokens, and RSA signatures with SHA-256. If you use any other algorithm, authentication will fail.

To create a new signing key, navigate to <Breadcrumb>Settings -> Key Master</Breadcrumb>, click on the <Breadcrumb>Generate EC key pair</Breadcrumb> dropdown button, and select the <Breadcrumb>Generate RSA key pair</Breadcrumb> option.

![Generate RSA key pair.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/generate-rsa-key.png)

Enter a name such as `Tableau RS256 Key` for the name of the signing key and save the changes.

![Name the signing key.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/name-signing-key.png)

## Create An Application

In your FusionAuth dashboard, navigate to <Breadcrumb>Applications</Breadcrumb> in the sidebar, then click the <Breadcrumb>+</Breadcrumb> button in the top right to create a new application. Give it a memorable name, such as `Tableau Cloud OIDC`, and set the tenant you want to use.

![Create a new application.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/create-application.png)

Before saving, navigate to the <Breadcrumb>OAuth</Breadcrumb> tab to configure the OAuth settings. Make sure the following conditions are met:

* **Authorized redirect URLs** is empty for now. You will retrieve its values from Tableau.
* Under **Enabled grants**, enable the following:
  * <Breadcrumb>Authorization Code</Breadcrumb>
  * <Breadcrumb>Refresh Token</Breadcrumb>

![Configure OAuth settings.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/oauth-configuration.png)

## Configure JWT Settings

Before saving the application, also navigate to the <Breadcrumb>JWT</Breadcrumb> tab, enable JWT, and configure the following fields:

* **Id token signing key**: Select the <Breadcrumb>RS256 key</Breadcrumb> you created earlier, in this case `Tableau RS256 Key`.
* **Access token signing key**: Select the same <Breadcrumb>RS256 key</Breadcrumb>.

![Configure JWT settings.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/jwt-configuration.png)

Save the application.

## Retrieve Application Credentials

After saving, you're taken back to the Applications listing page. To view the application details, click the dropdown in the **Action** column for your Tableau Cloud OIDC application and select <Breadcrumb>View</Breadcrumb>. From the application details page, retrieve the following values, which you'll need for configuring Tableau:

* The **Client Id** in the <Breadcrumb>OAuth configuration</Breadcrumb> section
* The **Client secret** in the <Breadcrumb>OAuth configuration</Breadcrumb> section
* The **OpenID Connect Discovery** URL in the <Breadcrumb>OAuth2 & OpenID Connect Integration details</Breadcrumb> section

![Application details with Client ID and secret.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/application-details.png)

## Configure Tableau Cloud

In the Tableau dashboard, navigate to <Breadcrumb>Settings -> Authentication</Breadcrumb>. Then, click on the <Breadcrumb>New Configuration</Breadcrumb> button and select <Breadcrumb>OIDC</Breadcrumb> for the authentication type and enter a memorable name for the configuration, such as `FusionAuth OIDC`.

![Create new OIDC configuration in Tableau.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/tableau-new-configuration.png)

Fill out the rest of the configuration form as follows:

* **Client ID**: Enter the **Client Id** you copied from FusionAuth.
* **Client Secret**: Enter the **Client secret**  you copied from FusionAuth.
* **Configuration URL**: Enter the **OpenID Connect Discovery** URL from the FusionAuth application.

![Enter OIDC configuration details.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/tableau-oidc-settings.png)

After entering the Client ID, Client Secret, and Configuration URL, scroll down to the section labeled <Breadcrumb>2. Copy Tableau URL redirect</Breadcrumb>. You'll see a **Redirect URL** that looks like `https://sso.online.tableau.com/public/oidc/login`. Copy this URL. You need it to configure FusionAuth in the next step.

### Update The FusionAuth Application With The Redirect URL

Return to your FusionAuth dashboard and navigate to <Breadcrumb>Applications</Breadcrumb>. In the Applications listing, click the edit (pencil) icon in the action buttons for your Tableau Cloud OIDC application. Go to the <Breadcrumb>OAuth</Breadcrumb> tab and paste the redirect URL you copied from Tableau into the **Authorized redirect URLs** field. Save the application.

### Configure Claims Mapping

Return to Tableau Cloud and continue with your OIDC configuration. In the Claims mapping section, configure the following:

* **Username**: Enter `email`.
* **Email address**: Enter `email`.
* **Name**: Select the <Breadcrumb>Given and family name</Breadcrumb> radio button.
  * **Given name or first name**: Enter `given_name`.
  * **Family name or last name**: Enter `family_name`.

![Configure claims mapping.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/claims-mapping.png)

Save the changes.

### Test The Configuration

Create a new FusionAuth user account for testing purposes.

Click <Breadcrumb>Test Configuration</Breadcrumb> to open a modal with the FusionAuth login screen. Fill in your test user credentials and submit the form. You should see a <Breadcrumb>Successfully connected to server</Breadcrumb> message below that button.

## Add A New User

In your Tableau Cloud account, navigate to <Breadcrumb>Users</Breadcrumb> in the sidebar. Click the <Breadcrumb>Add Users</Breadcrumb> dropdown and select <Breadcrumb>Add Users by Username</Breadcrumb>. You must provide the same email addresses that users will enter to log in to your FusionAuth instance.

Now, those users should receive an email with a link to the login page.

![Adding a user via email address.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/adding-user.png)

Users are redirected to the FusionAuth login screen when they click <Breadcrumb>Sign in</Breadcrumb>. There, each user fills in the credentials for their FusionAuth account. After users authenticate successfully, they are sent to the Tableau Cloud dashboard.

![Tableau dashboard.](/img/docs/lifecycle/authenticate-users/integrations/oidc/tableau/tableau-dashboard.png)

## Troubleshooting

Finally, you can edit your FusionAuth application. Go to the <Breadcrumb>JWT</Breadcrumb> and <Breadcrumb>OAuth</Breadcrumb> tabs and click <Breadcrumb>Debug enabled</Breadcrumb> to create an event log with more information. To see these logs, browse to <Breadcrumb>System -> Event Log</Breadcrumb>.

You can also review the [Tableau Cloud SSO docs](https://help.tableau.com/current/online/en-us/openid_auth.htm).
