[Salesforce](https://www.salesforce.com/) is a robust customer relationship management software that allows users to sign in into their accounts using an external Identity Provider. This document covers the configuration necessary to get Salesforce working with FusionAuth as the identity provider using OpenID Connect.

### Prerequisites

This document assumes you have a running instance of FusionAuth and a working Salesforce application. You will also need admin accounts for both to configure them correctly and a FusionAuth user that you will use to sign into Salesforce. You can use an existing user or create a new user for this purpose.

## Configure FusionAuth

Navigate to <Breadcrumb>Applications</Breadcrumb> and click on <Icon name="plus"/> to create an application named `Salesforce`.

![Create an Application.](/img/docs/lifecycle/authenticate-users/integrations/oidc/add-application.png)

Click on <Icon name="save"/> to save your application. After being redirected to the <Breadcrumb>Applications</Breadcrumb> page, click on the <Icon name="fa-search" /> icon to view the application details. Write down the **OAuth IdP login URL**, **Logout URL**, **Token endpoint** and **Userinfo endpoint** fields, located in the <Breadcrumb>OAuth2 & OpenID Connect Integration details</Breadcrumb> section, as you'll need that information later.

## Configure Salesforce

Before configuring FusionAuth as an identity provider in Salesforce, you need to create some code that will map the information returned from FusionAuth into a user entity that will then be persisted in Salesforce using [Apex](https://developer.salesforce.com/docs/atlas.en-us.apexcode.meta/apexcode/apex_intro_what_is_apex.htm), their object-oriented programming language.

Navigate to <Breadcrumb>PLATFORM TOOLS -> Custom Code -> Apex Classes</Breadcrumb> and click the <Breadcrumb>New</Breadcrumb> button to create a new class.

![Create an Apex class.](/img/docs/lifecycle/authenticate-users/integrations/oidc/apex-class-new.png)

In the editor, paste the following code as a starting point:

<RemoteCode url="https://raw.githubusercontent.com/FusionAuth/fusionauth-example-salesforce-oidc/main/RegistrationHandler.cls" />

Complete the code above by making the following changes:

- Update the `ORG_SUFFIX` to your actual FusionAuth instance URL
- Set `STANDARD_PROFILE` to the profile name you want to add users to (you can find the available options by navigating to <Breadcrumb>ADMINISTRATION -> Users -> Profiles</Breadcrumb>)
- Add a default time zone by setting `u.timeZoneSidKey` to a time zone code from [this list](https://help.salesforce.com/s/articleView?id=sf.admin_supported_timezone.htm&type=5). This will be used as a fallback when one is not received from FusionAuth.

Finally, click <Breadcrumb>Save</Breadcrumb> to create the class.

Browse to <Breadcrumb>SETTINGS -> Identity -> Auth. Providers</Breadcrumb> to get to the Authentication Providers page and click <Breadcrumb>New</Breadcrumb>.

![Create an Authentication Provider.](/img/docs/lifecycle/authenticate-users/integrations/oidc/auth-provider-new.png)

Fill in the values as shown in the image below, using the information you copied from your FusionAuth application earlier.

![Fill in Authentication Provider values.](/img/docs/lifecycle/authenticate-users/integrations/oidc/auth-provider-form.png)

The following table describes the necessary values that you need to copy from your FusionAuth application onto the fields in Salesforce.

| **Salesforce** field                                                         | **FusionAuth** field                                                                                         | Sample value                                                                                           |
|--------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------|
| **Provider Type**                                 |                                                                                                          | `Open ID Connect`                                                                                      |
| **Name**                                          |                                                                                                          | `FusionAuth`                                                              |
| **URL Suffix**                                    |                                                                                                          | `FusionAuth`                                                              |
| **Consumer Key**                                  | Your FusionAuth application **Client Id**                                         | `fb45c33d-72db-4852-b764-3e8556916193`                                                                 |
| **Consumer Secret**                               | Your FusionAuth application **Client secret**                                     | `QCuUxBdSq-B-GKpydrh63ASkHZnJZS7P-6Ib2PYZNQw`                                                          |
| **Authorize Endpoint URL**                        | Your FusionAuth application **OAuth IdP login URL** until the question mark (`?`) | `https://your.fusionauth.application.url/oauth2/authorize`                                             |
| **Token Endpoint URL**                            | Your FusionAuth application **Token endpoint**                                    | `https://your.fusionauth.application.url/oauth2/token`                                                 |
| **User Info Endpoint URL**                        | Your FusionAuth application **Userinfo endpoint**                                 | `https://your.fusionauth.application.url/oauth2/userinfo`                                              |
| **Token Issuer**                                  | Your FusionAuth application **Userinfo issuer**                                   | `https://your.fusionauth.application.url`                                                              |
| **Default Scopes**                                |                                                                                                          | `openid email`                                                                                         |
| **Send access token in header**                   |                                                                                                          | `Checked`                                                                                              |
| **Send client credentials in header**             |                                                                                                          | `Unchecked`                                                                                            |
| **Include Consumer Secret in SOAP API Responses** |                                                                                                          | `Unchecked`                                                                                            |
| **Custom Logout URL**                             | Your FusionAuth application **Logout Url**                                        | `https://your.fusionauth.application.url/oauth2/logout?client_id=fb45c33d-72db-4852-b764-3e8556916193` |

In **Registration Handler**, click the <Icon name="fa-search" /> magnifying glass button to open a window with the existing Apex Classes in your Salesforce organization. There, click <Breadcrumb>FusionAuthRegHandler</Breadcrumb> to select it. Click the other <Icon name="fa-search" /> magnifying glass button in **Execute Registration As** and select the user that will be responsible for executing the registration (you can select your own user here).

Click <Breadcrumb>Save</Breadcrumb> to finish configuring the provider. Scroll down to the <Breadcrumb>Salesforce Configuration</Breadcrumb> section and open the address from <Breadcrumb>Test-Only Initialization URL</Breadcrumb> in an incognito window.

![Copy Test-Only Initialization URL from the Authentication Provider.](/img/docs/lifecycle/authenticate-users/integrations/oidc/auth-provider-values.png)

After logging in with your FusionAuth credentials, you should be redirected to an XML file with the user details that Salesforce will receive from FusionAuth, like the example below.

```xml
<user>
    <full_name>Your Full Name</full_name>
    <provider>Open ID Connect</provider>
    <org_id>000000000000000</org_id>
    <last_name>Your Last Name</last_name>
    <id>00000000-0000-0000-0000-000000000000</id>
    <portal_id>000000000000000</portal_id>
    <first_name>Your First Name</first_name>
    <email>Your Email</email>
</user>
```

<Aside type="note">
If you are not seeing these values, click <Breadcrumb>Edit</Breadcrumb> in the Salesforce <Breadcrumb>Auth. Providers</Breadcrumb> page and double-check the values.
</Aside>

Now, go back to the <Breadcrumb>Salesforce Configuration</Breadcrumb> section in Salesforce and write down **Callback URL**, **Single Sign-On Initialization URL** and **Single Logout URL** values. In your FusionAuth instance, edit the <Breadcrumb>Salesforce</Breadcrumb> application you just created and paste **Callback URL** in **Authorized redirect URLs** and **Single Logout URL** in **Logout URL**.

Click on <Icon name="save"/> to save the changes.

![Editing the Application.](/img/docs/lifecycle/authenticate-users/integrations/oidc/edit-application.png)

## Log in

Open the **Single Sign-On Initialization URL** you copied from the <Breadcrumb>Salesforce Configuration</Breadcrumb> in a new browser or an incognito window. You should be taken to the FusionAuth login page. Fill in your credentials and you should be redirected back to your Salesforce account already logged in.

## Troubleshooting

Most errors occur due to misconfiguration of the Authentication Provider while copying values from FusionAuth into Salesforce. Make sure you have provided the right values there.
