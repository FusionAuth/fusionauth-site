Before importing users, create the roles they'll need in your FusionAuth application. For this example, create a "user" role.

Navigate to <Breadcrumb>Applications</Breadcrumb> and click the <Breadcrumb>Select</Breadcrumb> dropdown next to your test application. Select <Breadcrumb>Manage Roles</Breadcrumb>.

On the roles page, click the <Breadcrumb>Add</Breadcrumb> button (green plus sign) to create a new role.

Configure the role:

- Set <InlineField>Name</InlineField> to <InlineFieldValue>user</InlineFieldValue>.
- Enable the <Breadcrumb>Default</Breadcrumb> toggle so new users automatically receive this role.
- Leave the default values for the other settings.

Click <Breadcrumb>Submit</Breadcrumb> to create the role.

<LightboxImage src={`/img/docs/lifecycle/migrate-users/provider-specific/${props.migration_source_dir}/create-role.png`} alt="Configuring a role." />

You can create additional roles as needed based on your application's authorization structure. The export scripts reference these role names in the `registrations` section of the user data.
