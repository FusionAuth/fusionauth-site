Before importing users, create the roles they'll need in your FusionAuth application. For this example, create a "user" role.

Navigate to <Breadcrumb>Applications</Breadcrumb> and click the <InlineUIElement>Select</InlineUIElement> dropdown next to your test application. Select <InlineUIElement>Manage Roles</InlineUIElement>.

On the roles page, click the <InlineUIElement>Add</InlineUIElement> button (green plus sign) to create a new role.

Configure the role:

- Set <InlineField>Name</InlineField> to `user`.
- Enable the <InlineUIElement>Default</InlineUIElement> toggle so new users automatically receive this role.
- Leave the default values for the other settings.

Click <InlineUIElement>Submit</InlineUIElement> to create the role.

<img src={`/img/docs/lifecycle/migrate-users/provider-specific/${props.migration_source_dir}/create-role.png`} alt="Configuring a role." width="1200" />

You can create additional roles as needed based on your application's authorization structure. The export scripts reference these role names in the `registrations` section of the user data.
