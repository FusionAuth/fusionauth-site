It is best to create a separate tenant for migration testing. Tenants logically isolate configuration settings and users. If a migration goes awry or you need to redo it after tweaking settings, you can delete the test tenant and start with a clean system. To add a tenant, navigate to <strong>Tenants</strong> and choose the <InlineUIElement>Add</InlineUIElement> button (green plus sign).

<img src={`/img/docs/lifecycle/migrate-users/provider-specific/${props.migration_source_dir}/list-of-tenants-add-highlighted.png`} alt="Adding a tenant." width="1200" role="bottom-cropped" />

Give it a descriptive <InlineField>Name</InlineField> like <code>{props.migration_source_name} import test</code>. You shouldn't need to modify any of the other configuration options to test importing users.

Save the tenant.

<img src={`/img/docs/lifecycle/migrate-users/provider-specific/${props.migration_source_dir}/add-tenant.png`} alt="The tenant creation screen." width="1200" role={props.add_tenant_image_role} />

Record the Id of the tenant, which will be a UUID. It will look something like `25c9d123-8a79-4edd-9f76-8dd9c806b0f3`. You'll use this later.

<img src={`/img/docs/lifecycle/migrate-users/provider-specific/${props.migration_source_dir}/list-of-tenants.png`} alt="The tenant list." width="1200" role="bottom-cropped" />
