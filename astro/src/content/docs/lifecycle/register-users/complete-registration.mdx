Complete registration allows you to redirect existing registered users to `/oauth2/complete-registration` when required profile fields are missing, without allowing new visitors to create accounts. This is distinct from self-service registration, which enables account creation.

## What Is Complete Registration

When complete registration is enabled on an application, FusionAuth checks whether a registered user's profile satisfies all required fields after they authenticate. If any required fields are missing, FusionAuth redirects the user to `/oauth2/complete-registration` to collect the missing information before sending them to the application's redirect URI.

Complete registration and self-service registration are independent settings. You can enable complete registration without enabling self-service registration, and vice versa.

## Configuring Complete Registration

<AvailableSince since="1.65.0"/>

<Aside type="note">
Complete registration is available on all plans, including Community.
</Aside>

To enable complete registration for an application:

1. Navigate to <Breadcrumb>Applications</Breadcrumb> and open the application you want to configure.
1. Click the <Breadcrumb>Registration</Breadcrumb> tab.
1. Enable the <InlineField>Complete registration</InlineField> toggle.
1. Configure the registration form. The form settings work the same way as they do for self-service registration. For guidance on creating and configuring forms, see [Advanced Registration Forms](/docs/lifecycle/register-users/advanced-registration-forms).
1. Save the application.

## The Complete Registration Flow

When a user authenticates with an application that has complete registration enabled:

1. FusionAuth checks whether the user's profile satisfies all required fields defined on the registration form.
1. If all required fields are present, the user proceeds normally to the application's redirect URI.
1. If any required fields are missing, FusionAuth redirects the user to `/oauth2/complete-registration`.
1. The user completes the form and submits.
1. FusionAuth sends the user to the application's redirect URI.

<Aside type="note">
Email and phone verification steps on registration forms are skipped for existing users in this flow, since the user's account already exists.
</Aside>
