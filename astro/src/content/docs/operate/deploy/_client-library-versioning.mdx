FusionAuth represents releases with a version number: `<major>.<minor>.<patch>`. Major and minor releases can change the API, but patch releases never change the API.

Client library version numbers show you which API version a client library was built for. Always use a client library with the same `major` and `minor` numbers as your FusionAuth instance version. This ensures that your client library uses the same API version as your FusionAuth instance.

Patch releases don't always exist for client libraries since many bugfixes and small changes don't impact client libraries. FusionAuth only creates patch releases for client libraries when we fix an issue in the library, so always upgrade to the latest `patch` release for your `major` and `minor` version.

For example, if your FusionAuth instance runs version `1.61.2`:

* use Java client library version [`1.61.2`](https://github.com/FusionAuth/fusionauth-java-client/releases/tag/1.61.2), the latest patch release for `1.61.x`.
* use TypeScript client library version [`1.61.0`](https://www.npmjs.com/package/@fusionauth/typescript-client/v/1.61.0), the latest patch release for `1.61.x`
