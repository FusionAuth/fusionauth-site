## Overview

[Datadog](https://docs.datadoghq.com/getting_started/application/) is an observability service you can integrate with your FusionAuth applications to collect events and metrics and send them to Datadog to analyze your monitoring and performance data.

In this guide, you will go through how to connect FusionAuth to Datadog. The guide will cover.

- How to create and set up a Datadog account.
- How to add a Datadog Agent inside FusionAuth Docker image to collect and export telemetry data.
- How to create a dashboard to monitor FusionAuth metrics.
- How to add a Datadog Agent on a separate (windows) host and monitor the Prometheus endpoint exposed by FusionAuth.
- How to set up an OpenTelemetry Linux collector to receive, process, and export telemetry data.
- How to add export logs or events with a Python app with the use of the Datadog API.

## FusionAuth Metrics

Before continuing with this guide, if you haven't already please go through the [FusionAuth guide to monitoring](/docs/operate/secure-and-monitor/monitor) to get an overview of what metrics are available and choose the ones relevant to your needs.

### Set Up Prometheus API Access In FusionAuth

FusionAuth exposes metrics in a "Prometheus Endpoint". To use the endpoint you need to generate an API key in FusionAuth and grant "GET" request permission to the API key to access the endpoint. To do this:

- Login to your FusionAuth instance and navigate to **Settings -> API keys**.
- On the top right side of the page, click on the <IconButton icon="plus" color="green" /> button to add a new API key.

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-fusionauth-enableapikey-prometheus-endpoint.png" alt="Enable API Key prometheus endpoint FusionAuth" role="bottom-cropped" width="1200" />

- Enter a <InlineField>Description</InlineField> for the API Key.
- Scroll down and enable the "GET" permission on the `/api/prometheus/metrics` endpoint.

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-enableget-prometheus-endpoint.png" alt="Enable API Key prometheus endpoint FusionAuth" role="bottom-cropped" width="1200" />

- Click on the <IconButton icon="save" color="blue" /> button to save the API Key.
- After saving the API key click on the red lock <IconButton icon="lock" color="red" />, next to the key that you generated to reveal and copy the value of the key.
- Store this key, as you will need it later in the guide.

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-fusionauth-apikey.png" alt="FusionAuth API Key" role="bottom-cropped" width="1200" />

### Datadog Agent Set Up Using Docker

The Datadog Agent collects metrics and events from your system and apps. It can be installed anywhere, even on your workstation. If you select the Ubuntu agent, it will show the agent installation code below that uses a script to handle the installation of the agent. If you prefer a more manual approach to install the agent you can expand the <InlineUIElement>Manual Step by Step Instructions</InlineUIElement> instructions.

```bash
DD_API_KEY=<yourapikey> DD_SITE="us5.datadoghq.com" bash -c "$(curl -L https://s3.amazonaws.com/dd-agent/scripts/install_script_agent7.sh)"
```

<img src="/img/docs/operate/secure-and-monitor/datadog/install-datadog-agent-ubuntu.png" alt="Install Datadog Agent Ubuntu" role="bottom-cropped" width="1200" />

The next step will then be to build a FusionAuth Docker image that has the agent installed using the code above.

To build a FusionAuth Docker image save the Dockerfile from the [FusionAuth containers repo](https://github.com/FusionAuth/fusionauth-containers/blob/master/docker/fusionauth/fusionauth-app/Dockerfile) to your computer. Edit the Dockerfile file and insert the following lines above the comment labelled "###### Start FusionAuth App".

```
### NEW FOR DATADOG ###
RUN mkdir -p /var/lib/apt/lists/partial \
&& chmod 755 /var/lib/apt/lists/partial \
&& apt update \
&& apt install -y ca-certificates curl  sudo \
&& cd /usr/local/fusionauth \
&& curl -L -o install_script_agent7.sh https://s3.amazonaws.com/dd-agent/scripts/install_script_agent7.sh \
&& chmod +x /usr/local/fusionauth/install_script_agent7.sh

### Install DATADOG Agent App ###
ENV DD_API_KEY="<your API key from datadog website>"
ENV DD_SITE="us5.datadoghq.com" #<your site from Datadog website>
ENV DD_INSTALL_ONLY=false
ENV DD_HOSTNAME="fusionauthdocker"
RUN /bin/bash -c "/usr/local/fusionauth/install_script_agent7.sh"

### Copy the OpenMetrics configuration file ###
COPY openmetrics.d-conf.yaml /etc/datadog-agent/conf.d/openmetrics.d/conf.yaml

# Allow fusionauth user to run datadog-agent service without password
RUN echo "fusionauth ALL=(ALL) NOPASSWD: /usr/sbin/service datadog-agent start" > /etc/sudoers.d/fusionauth

# Ensure correct ownership and permissions for the log directory
RUN mkdir -p /usr/local/fusionauth/logs \
    && chown -R fusionauth:fusionauth /usr/local/fusionauth/logs

# # Modify FusionAuth start script to start Datadog Agent as root and continue on failure
# RUN head -n -1 /usr/local/fusionauth/fusionauth-app/bin/start.sh > temp.sh \
# && echo 'sudo service datadog-agent start <&- >>  /dev/null' >> temp.sh \
# && mv temp.sh /usr/local/fusionauth/fusionauth-app/bin/start.sh \
# && chown fusionauth:fusionauth /usr/local/fusionauth/fusionauth-app/bin/start.sh \
# && chmod +x /usr/local/fusionauth/fusionauth-app/bin/start.sh
```


This configuration will do the following:

- It downloads the Datadog Agent from https://s3.amazonaws.com/dd-agent/scripts/install_script_agent7.sh and sets environment variables to complete the installation.
- After installing the Datadog Agent, the config `openmetrics.d-conf.yaml` is copied into the image. This configuration file will handle the OpenMetrics integration.

Create the  `openmetrics.d-conf.yaml` file in the same folder as the Dockerfile. Add the following configuration to the `openmetrics.d-conf.yaml` file:

 ```yaml
init_config:

instances:
  - prometheus_url: http://localhost:9011/api/prometheus/metrics
    namespace: FusionAuth_docker_metrics_openmetric_agent
    metrics:
      - '*'
    headers:
      Authorization: '<Your fusion Auth API key with access to Prometheus metrics>'
      Content-Type: 'application/json'
```

Build the Dockerfile into a new image to use in place of the official FusionAuth image.

```sh
docker build --platform linux/amd64 -t faimage .
```

To start the necessary FusionAuth Docker containers first save the `docker-compose.yaml` file from the [FusionAuth containers repo](https://github.com/FusionAuth/fusionauth-containers/blob/main/docker/fusionauth/docker-compose.yml) and the sample [`.env`](https://github.com/FusionAuth/fusionauth-containers/blob/main/docker/fusionauth/.env) file. Change the line `image: fusionauth/fusionauth-app:latest` in the `docker-compose.yaml` file to point to the image that you have just built `image: faimage:latest` and run the following command in a terminal where the `docker-compose.yaml` file is saved to  start the services.

```sh
docker compose up -d
```

<Aside type="note">
  The code to autostart the `datadog-agent` is commented out in the sample Dockerfile configuration because the current version of the agent gives a `fail message` when it starts, although it starts successfully.
  This stops FusionAuth from starting in the docker container. You need to start the agent manually when FusionAuth is running with the following commands.

  ```sh
    docker ps   # to get the ID of the FusionAuth container which should look something like d5a0c674288a

    docker  exec -u root -it d5a0c674288a service datadog-agent start  #start the agent

    docker  exec -u root -it d5a0c674288a service datadog-agent status #get the service status should respond with  * datadog-agent is running
  ```
</Aside>

When the agent is successfully installed and started in the FusionAuth container if you refresh the window on the Datadog website, it will indicate that metrics from the agent are being received.

### Datadog Agent On A Remote Host To Capture Prometheus/OpenMetrics Data

The Datadog Agent can be installed on a separate machine to monitor Prometheus metrics or OpenMetrics data from a remote FusionAuth host.


FusionAuth exposes a Prometheus endpoint, which can be monitored by an "OpenMetric" Integration in Datadog. To access this endpoint you must configure an API key to allow Prometheus metrics on the FusionAuth instance. See [Setup Prometheus API access in FusionAuth](#set-up-prometheus-api-access-in-fusionauth).

To install the OpenMetrics integration in Datadog select <strong>Integrations</strong> on the bottom left of the left sidebar. Search for "OpenMetrics", and click on the tile.

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-openmetrics-install-integration.png" alt="Datadog OpenMetrics integration" role="bottom-cropped" width="1200" />

The installation consists of 2 parts. First you install the integration using the install button on the tile and then you will need to install the Datadog Agent and configure an OpenMetrics or Prometheus collector.

The installation steps for the Prometheus integration are similar to the above but instead of searching for OpenMetrics, search for [Prometheus](https://us5.datadoghq.com/integrations/prometheus?search=Prometheus).

To install the Datadog Agent navigate to the agent download [page](https://us5.datadoghq.com/account/settings/agent/latest?platform=overview).

- Select your platform. Make sure of the region. The guide uses "us5.datadoghq.com", but yours might vary depending on the selection when you created your Datadog [account](#datadog-account-set-up).
- Download the agent installer using the instructions on the page. If prompted, choose a folder to save the download, else it will be saved in your default downloads folder.
- Run the downloaded installer.
- Follow the prompts, accept the license agreement, and enter your Datadog API key and region
- When the installation finishes, you are given the option to launch the Datadog Agent Manager. Launch the Datadog Agent Manager

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-agent-windows-setup.png" alt="Datadog agent set up" role="bottom-cropped" width="1200" />

- Look for a green button at the top that says "Connected to Agent". If not, follow the installation instructions again.
- Click on <InlineUIElement>Checks</InlineUIElement> on the left sidebar and select <InlineUIElement>Manage Checks</InlineUIElement>. Select "prometheus" from the list.

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-agent-prometheus-config.png" alt="Datadog agent prometheus config" role="bottom-cropped" width="1200" />

Complete the following config for the remote FusionAuth server, in the right-hand pane:

```yaml
## All options defined here are available to all instances.
init_config:
  
instances:
  - prometheus_url: https://<your fusionauth host>/api/prometheus/metrics
    namespace: Prometheus_remote_metrics #namespace to identify requests in datadog
    metrics:
      - '*'  #you can filter or add specific metrics here , see documentation.
    headers:
      Authorization: '<your FusionAuth apikey>'
      Content-Type: 'application/json'
```

When the config is done click on <InlineUIElement>Add check</InlineUIElement> to save.
Restart the agent. You can confirm on the Checks summary screen if everything is running ok.

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-agent-prometheus-check-summary.png" alt="Datadog agent prometheus check summary" role="bottom-cropped" width="1200" />

To see the results in Datadog navigate to the <strong>Metrics Explorer</strong> on the Datadog website. You can use the namespace defined above in the config `Prometheus_remote_metrics` to add metrics in the metrics explorer. Choose the last 15 minutes for the period. You will see data being pulled from the  FusionAuth  Prometheus endpoint. Following the [Datadog Dashboard Set Up](#datadog-dashboard-set-up) steps, you can easily add these metrics to your dashboards as required.

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-website-metrics-explorer-prometheus.png" alt="Datadog website metrics explorer prometheus" role="bottom-cropped" width="1200" />


### Sending Events With A Python App To Datadog

Datadog also provides an API whereby you can create cron jobs or services that run on your FusionAuth instance that provide additional/custom events or logs to Datadog.

Below is a sample python application that just checks if a given FusionAuth host is alive and sent the status event to Datadog. Run the following command before you try the script:

```sh
pip install requests datadog-api-client
```

```python
import requests

from datadog_api_client import ApiClient, Configuration
from datadog_api_client.v1.api.events_api import EventsApi
from datadog_api_client.v1.model.event_create_request import EventCreateRequest
from datetime import datetime

# Get the current date and time
now = datetime.now()


def is_website_alive(url):
    try:
        response = requests.get(url, timeout=10)
        # Check if the response status code is 200 (OK)
        if response.status_code == 200:
            return True
        else:
            return False
    except requests.RequestException as e:
        # If there was an issue (e.g., network error, timeout), the website is not reachable
        print(f"Error: {e}")
        return False
    

formatted_now = now.strftime("%Y-%m-%d %H:%M:%S")


url = "https://<your fusionauth/>"
if is_website_alive(url):
    status  = f"[{ formatted_now}] - {url} is alive. "
else:
    status = f"[{ formatted_now}] - {url} is not reachable."

#### here you can define your message/event
body = EventCreateRequest(
    title="FusionAuthStatusEvent",
    text=status,
    tags=[
        "FusionAuth:StatusEvent",
    ],
)



configuration = Configuration()

configuration.api_key["apiKeyAuth"] = "<your DataDog API KEY>"
configuration.server_variables["site"] = "us5.datadoghq.com" #your Region
configuration.api_key["appKeyAuth"] = "<your Application key if needed>"

with ApiClient(configuration) as api_client:
    api_instance = EventsApi(api_client)
    response = api_instance.create_event(body=body)

    print(response)
```

After running the script navigate to <strong>Service Mgmt -> Event management -> Explorer </strong> on the Datadog website. You will see the event that was logged with the script.

<img src="/img/docs/operate/secure-and-monitor/datadog/datadog-custom-events.png" alt="Datadog custom events" role="bottom-cropped" width="1200" />

Read more about all the possible Datadog API's [here](https://docs.datadoghq.com/api/latest/using-the-api/).

### Sending Data To Datadog With The OpenTelemetry Collector

You can also use the OpenTelemetry Java collector to push data from the FusionAuth host to Datadog.

To use OpenTelemetry Java collector with Docker you need to modify the official FusionAuth Docker image to download the OpenTelemetry Java agent and change the script that starts FusionAuth. Configuration values for Java OpenTelemetry are described [here](https://opentelemetry.io/docs/languages/java/configuration).

Save the Dockerfile from the [FusionAuth containers repo](https://github.com/FusionAuth/fusionauth-containers/blob/master/docker/fusionauth/fusionauth-app/Dockerfile) to your computer.

Edit the file. Above the comment labelled "###### Start FusionAuth App", insert the code below.

```
##### NEW FOR OPENTELEMETRY #######################################################################
RUN mkdir -p /var/lib/apt/lists/partial \
    && chmod 755 /var/lib/apt/lists/partial \
    && apt update \
    && apt install -y ca-certificates \
    && cd /usr/local/fusionauth \
    && curl -L -o otel.jar https://github.com/open-telemetry/opentelemetry-java-instrumentation/releases/download/v2.4.0/opentelemetry-javaagent.jar\
    && (head -n -1 /usr/local/fusionauth/fusionauth-app/bin/start.sh; echo 'exec "${JAVA_HOME}/bin/java" -javaagent:/usr/local/fusionauth/otel.jar -Dotel.resource.attributes=service.name=fusionauth -Dotel.traces.exporter=otlp -Dotel.exporter.otlp.endpoint=http://otel:4318 -cp "${CLASSPATH}" ${JAVA_OPTS} io.fusionauth.app.FusionAuthMain <&- >> "${LOG_DIR}/fusionauth-app.log" 2>&1') > temp.sh \
    && mv temp.sh /usr/local/fusionauth/fusionauth-app/bin/start.sh;
RUN chown fusionauth:fusionauth /usr/local/fusionauth/otel.jar /usr/local/fusionauth/fusionauth-app/bin/start.sh \
    && chmod +x /usr/local/fusionauth/fusionauth-app/bin/start.sh
```

- This code edits the `start.sh` file, the script that starts FusionAuth when the container starts.
- The command takes the head of the file (all lines except the last), and writes the new command to the end of the file.
- By default, the OpenTelemetry Java agent sends data to the OpenTelemetry collector at http://localhost:4317. The code above changes it to send data to the container, at http://otel:4318.

Build the Dockerfile into a new image to use in place of the official FusionAuth one.

```sh
docker build --platform linux/amd64 -t faimage .
```

To start the necessary FusionAuth Docker containers first save the `docker-compose.yaml` file from the [FusionAuth containers repo](https://github.com/FusionAuth/fusionauth-containers/blob/main/docker/fusionauth/docker-compose.yml) and the sample [`.env`](https://github.com/FusionAuth/fusionauth-containers/blob/main/docker/fusionauth/.env) file. Change the line `image: fusionauth/fusionauth-app:latest` in the `docker-compose.yaml` file to point to the image that you have just built `image: faimage:latest`.

Add the following service to `docker-compose.yaml` file. This will include the OpenTelemetery collector service in your compose configuration.

``` yaml
# OpenTelemetry Collector Contrib
  otel:
    image: otel/opentelemetry-collector-contrib:latest
    volumes:
      - ./otel-collector.yml:/etc/otel-collector.yml
    command: ["--config=/etc/otel-collector.yml"]
    #,"--feature-gates=-exporter.datadogexporter.DisableAPMStats"]
    ports:
      - "1888:1888"   # pprof extension
      - "8888:8888"   # Prometheus metrics exposed by the collector
      - "8889:8889"   # Prometheus exporter metrics
      - "13133:13133" # health_check extension
      - "4317:4317"   # OTLP gRPC receiver
      - "4318:4318"   # OTLP gRPC receiver
      - "55679:55679" # zpages extension
    environment:
      - OTEL_EXPORTER_OTLP_LOGS_ENDPOINT=http://otel:4318/v1/logs
      - OTEL_EXPORTER_OTLP_METRICS_ENDPOINT=http://otel:4318
      - OTEL_EXPORTER_OTLP_TRACES_ENDPOINT=http://otel:4318/v1/traces
    networks:
      - db_net
      
    depends_on:
      - db
      - search
      - fusionauth
```

Create a `otel-collector.yml` file in the same directory as the `docker-compose.yaml` file, for use with the otel container. This file is passed into the container with the `docker-compose.yaml` configuration.

```yaml
receivers:
  hostmetrics:
      scrapers:
        load:
        cpu:
        disk:
        filesystem:
        memory:
        network:
        paging:
        # process:
  otlp:
    protocols:
      http:
        #endpoint: "localhost:4318"
      grpc:
        #endpoint: "localhost:4317"
processors:
  batch:
    send_batch_max_size: 100
    send_batch_size: 10
    timeout: 10s
    
connectors:
  datadog/connector:

exporters:
  datadog/exporter:
    api:
      site: "us5.datadoghq.com"   #<make sure of your region>
      key: "<your Datadog API key>" 
service:
  pipelines:
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [datadog/connector, datadog/exporter]
    metrics:
      receivers: [hostmetrics, otlp, datadog/connector] #// The connector provides the metrics to your metrics pipeline
      processors: [batch]
      exporters: [datadog/exporter]
```

Start the Docker containers with the command below:
  
```bash
docker compose -f docker-compose.yaml up -d

#if you want to monitor the progress
# docker compose logs --follow
```

## Conclusion

The guide looked at various ways to get data into Datadog from a FusionAuth instance.

- The Datadog Agent can be installed and configured with various integrations inside the FusionAuth docker container.
- The Datadog Agent can be installed on an external host to monitor FusionAuth Prometheus Endpoint.
- You looked at using the Datadog API with a Python app to push sample events/logs to Datadog.

You can visualize collected metrics in dashboards created on the Datadog website. It is fair to say many metrics are available from FusionAuth to Datadog. Should you need any others, you can create your own using the API.
