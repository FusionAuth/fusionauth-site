<Aside type="note">
The email templates ship with a base URL of `http://localhost:9011` for links, such as the verify email or change password link. While this works for local development, for production, make sure to update all base URLs to a public URL, such as `https://auth.example.com`.

One option is to store a base URL value on the <InlineField>tenant.data</InlineField> object. For example, set <InlineField>tenant.data.baseEmailURL</InlineField> to <InlineFieldValue>https://auth.example.com</InlineFieldValue>, and then access it from each email template. Setting the <InlineField>tenant.data.baseEmailURL</InlineField> field can be done using the API or the client libraries.

There is currently no support for changing the base URL in all templates in an instance, though there is an [open feature request](https://github.com/FusionAuth/fusionauth-issues/issues/457).
</Aside>
