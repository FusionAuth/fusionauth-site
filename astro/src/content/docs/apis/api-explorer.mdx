The [FusionAuth API Explorer](/docs/apis/api-explorer) is an interactive tool that uses the [FusionAuth OpenAPI specification](https://github.com/fusionauth/fusionauth-openapi) to let you explore and test FusionAuth APIs directly in your browser. It is designed to be used against a local FusionAuth instance.

## Prerequisites

FusionAuth can be locally installed or used as a SaaS solution. The API explorer is useful only for a local installation.

[Install FusionAuth locally](/docs/get-started/download-and-install) before using the explorer.

After you install FusionAuth, [create an API key with the correct permissions](/docs/apis/authentication#managing-api-keys). Enter it in the **Authentication** section at the top of the explorer. Your API key will not be stored between sessions.

## CORS setup

To make in-browser API calls from the explorer, you need to configure CORS on your local FusionAuth instance. This is not required if you only use the generated curl commands to call the API from the command line.

To set up CORS:

1. Log in to your FusionAuth administrative user interface.
2. Navigate to <Breadcrumb>Settings -> System</Breadcrumb>.
3. Enable **CORS**.
4. Update **Allowed headers** to include `authorization`, `accept`, and `content-type`.
5. Check all the **Allowed methods**.
6. Set **Allowed origins** to your origin, or `*` to allow any origin.

<Aside type="caution">Do not run your production FusionAuth instance with CORS configured this openly. For local development and exploration this is acceptable.</Aside>

## Using the explorer

Once you have entered your API key, you can browse and test any FusionAuth API endpoint:

- Select an endpoint from the sidebar to see its parameters and request body schema.
- Fill in the required fields and click **Send** to make a live request against your local instance.
- Use the **Client Libraries** panel to get generated code examples in Shell, Ruby, Node.js, PHP, Python, and more.
- Deep links work natively. You can share a URL that opens directly to a specific endpoint.
- Use the **YAML** or **JSON** links in the toolbar to download the full spec.

## Versioning

The explorer loads the latest FusionAuth API spec by default. To test against an older version:

- Use the **Version** dropdown at the top of the explorer to select from available releases.
