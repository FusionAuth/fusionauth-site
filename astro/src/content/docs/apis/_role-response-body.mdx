#### Response Body

<APIBlock>
  <APIField name="role.description" type="String">
    The description of the role.
  </APIField>
  <APIField name="role.id" type="UUID">
    The Id of the Role.
  </APIField>
  <APIField name="role.insertInstant" type="Long">
    The [instant](/docs/reference/data-types#instants) that the Role was added to the FusionAuth database.
  </APIField>
  <APIField name="role.lastUpdateInstant" type="Long">
    The [instant](/docs/reference/data-types#instants) that the Role was updated in the FusionAuth database.
  </APIField>
  <APIField name="role.name" type="String">
    The name of the Role.
  </APIField>
  <APIField name="role.isDefault" type="Boolean">
    Whether or not the Role is a default role. A default role is automatically assigned to a user during registration if no roles are provided.
  </APIField>
  <APIField name="role.isSuperRole" type="Boolean">
    Whether or not the Role is a considered to be a super user role. This is a marker to indicate that it supersedes all other roles. FusionAuth will attempt to enforce this contract when using the web UI, it is not enforced programmatically when using the API.
  </APIField>
</APIBlock>
