#### Request Body

<APIBlock>
  <APIField name="integration.cleanSpeakConfiguration.apiKey" type="String" optional>
    The API key that is used when calling CleanSpeak for moderation.
  </APIField>
  <APIField name="integration.cleanSpeakConfiguration.enabled" type="Boolean" optional defaults="false">
    True if CleanSpeak integration is enabled.
  </APIField>
  <APIField name="integration.cleanSpeakConfiguration.url" type="String" optional>
    The URL of the CleanSpeak WebService service (i.e. http://localhost:8001/). Required when the CleanSpeak integration is enabled.
  </APIField>
  <APIField name="integration.cleanSpeakConfiguration.usernameModeration.applicationId" type="UUID" optional>
    The Id of the CleanSpeak Application where usernames are sent for moderation.
  </APIField>
  <APIField name="integration.cleanSpeakConfiguration.usernameModeration.enabled" type="Boolean" optional defaults="false">
    True if CleanSpeak username moderation is enabled.
  </APIField>
  <APIField name="integration.kafka.defaultTopic" type="String" optional>
    The name of the Kafka topic to send messages. Required when the Kafka integration is enabled.
  </APIField>
  <APIField name="integration.kafka.enabled" type="Boolean" optional defaults="false">
    True if the Kafka integration is enabled.
  </APIField>
  <APIField name="integration.kafka.producer" type="Map<String, String>" optional>
    String key value pairs to be used when building the Kafka Producer.
  </APIField>
  <APIField name="integration.twilio.accountSID" type="String" optional deprecated>
    The Twilio Account ID to use when connecting to the Twilio API. This can be found in your Twilio dashboard. Required when the Twilio integration is enabled.

    <RemovedSince since="1.26.0">
    In version 1.26.0 and beyond, Twilio configuration can be managed in the [Messengers API](/docs/apis/messengers/).
    </RemovedSince>
</APIField>
  <APIField name="integration.twilio.authToken" type="String" optional deprecated>
    The Twilio Auth Token to use when connecting to the Twilio API. This can be found in your Twilio dashboard. Required when the Twilio integration is enabled.

    <RemovedSince since="1.26.0">
      In version 1.26.0 and beyond, Twilio configuration can be managed in the [Messengers API](/docs/apis/messengers/).
    </RemovedSince>
  </APIField>
  <APIField name="integration.twilio.enabled" type="Boolean" optional defaults="false" deprecated>
    True if the Twilio integration is enabled.

    <RemovedSince since="1.26.0">
      In version 1.26.0 and beyond, Twilio configuration can be managed in the [Messengers API](/docs/apis/messengers/).
    </RemovedSince>
  </APIField>
  <APIField name="integration.twilio.fromPhoneNumber" type="String" optional deprecated>
    The configured Twilio phone number that will be used to send messages. This can be found in your Twilio dashboard.

    <RemovedSince since="1.26.0">
      In version 1.26.0 and beyond, Twilio configuration can be managed in the [Messengers API](/docs/apis/messengers/).
    </RemovedSince>
  </APIField>
  <APIField name="integration.twilio.messagingServiceSid" type="String" optional deprecated>
    The Twilio message service Id, this is used when using Twilio Copilot to load balance between numbers. This can be found in your Twilio dashboard.

    When using the Twilio Messaging Services Id, you may omit the fromPhoneNumber field.

    <RemovedSince since="1.26.0">
      In version 1.26.0 and beyond, Twilio configuration can be managed in the [Messengers API](/docs/apis/messengers/).
    </RemovedSince>
  </APIField>
</APIBlock>
