#### Request Body

<APIBlock>
  <APIField name="entity.data" type="Object" optional>
    An object that can hold any information about the Entity that should be persisted. Please review the [limits on data field types](/docs/get-started/core-concepts/limitations#data-type-changes-in-data-fields) as you plan for and build your custom data schema.
  </APIField>
  <APIField name="entity.clientId" type="String" optional>
    The OAuth 2.0 client Id. If you leave this blank during a POST, the value of the Entity Id will be used.
  </APIField>
  <APIField name="entity.clientSecret" type="String" optional>
    The OAuth 2.0 client secret. If you leave this blank during a POST, a secure secret will be generated for you. If you leave this blank during PUT, the previous value will be maintained. For both POST and PUT you can provide a value and it will be stored.
  </APIField>
  <APIField name="entity.name" type="String" required>
    A descriptive name for the Entity (i.e. "Raviga" or "Email Service").
  </APIField>
  <APIField name="entity.type.id" type="UUID" required>
    The Id of the Entity Type. Types are consulted for permission checks.
  </APIField>
</APIBlock>
