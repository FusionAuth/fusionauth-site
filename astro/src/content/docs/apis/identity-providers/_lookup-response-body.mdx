#### Example Response JSON

<APIBlock>
  <APIField name="identityProvider.applicationIds" type="Array<UUID>" since="1.16.0">
    The list of Application Ids that are enabled for this Identity Provider.
  </APIField>
  <APIField name="identityProvider.id" type="UUID">
    The unique Id of the Identity Provider.
  </APIField>
  <APIField name="identityProvider.idpEndpoint" type="String" since="1.16.0">
    The configured SAML v2 endpoint. This value will only be returned for a SAML v2 Identity Provider. This may be used to aid in building the SAML request so that it does not need to be hard coded in your application.
  </APIField>
  <APIField name="identityProvider.insertInstant" type="Long">
    The [instant](/docs/reference/data-types#instants) that the provider was added to the FusionAuth database.
  </APIField>
  <APIField name="identityProvider.lastUpdateInstant" type="Long">
    The [instant](/docs/reference/data-types#instants) that the provider was updated in the FusionAuth database.
  </APIField>
  <APIField name="identityProvider.name" type="String">
    The name of the Identity Provider.
  </APIField>
  <APIField name="identityProvider.oauth2.authorization_endpoint" type="String">
    The OAuth2 Authorize endpoint. This may be used to as the redirect location to begin the authorize workflow so that it does not need to be hard coded in your application.
  </APIField>
  <APIField name="identityProvider.oauth2.token_endpoint" type="String">
    The OAuth2 Token endpoint. This may be used during your integration so that this URI does not need to be hard coded in your application.
  </APIField>
  <APIField name="identityProvider.tenantId" type="UUID" since="1.62.0">
    The Id of the Tenant that this Identity Provider belongs to for tenant-scoped identity providers. The value is omitted on the response for global identity providers.
  </APIField>
</APIBlock>
