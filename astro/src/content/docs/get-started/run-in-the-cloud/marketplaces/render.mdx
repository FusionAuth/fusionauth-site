You can run FusionAuth on most cloud providers, including [Render](https://render.com/), a platform for deploying containerized applications.

## Prerequisites

To complete this guide, you will need:

* **A Render account**: for this tutorial, you can use the free version, but a production deployment requires a paid account. Performance on the free tier will be much slower than on a paid account.

## Fork the render blueprint

Render provides the ability to set up projects with **blueprints**, a form of Infrastructure as Code (IaC) based on a YAML file. The FusionAuth blueprint includes a PostgreSQL database and a FusionAuth Web service.

While you can run a blueprint directly from our repository, it’s a best practice to run this blueprint from a forked version of the repository in your own GitHub account. Since blueprints automatically update when the repository changes, you must own your blueprint repository to maintain full control over your instance.

To create your own fork based on the FusionAuth blueprint:

1. Visit the [FusionAuth/fusionauth-render-blueprint repository](https://github.com/FusionAuth/fusionauth-render-blueprint).
1. Click the <InlineUIElement>Fork</InlineUIElement> button in the upper right.
1. Use the default values and click <InlineUIElement>Create fork</InlineUIElement> to create your fork.
1. Wait a few seconds for GitHub to create your fork.
1. Once your browser loads a new repository with the subtitle "forked from FusionAuth/fusionauth-render-blueprint", you're ready to configure your instance.

## Create a new blueprint

1. On your Render account dashboard, click the <InlineUIElement>Blueprints</InlineUIElement> link in the navigation.

1. Click <InlineUIElement>New Blueprint Instance</InlineUIElement>.

1. Connect your repository using the URL, which should look something like: `https://github.com/<your-github-username>/fusionauth-render-blueprint`.

1. Set up your project with the following configuration:
   * name: `FusionAuth`
   * branch: `main`
   * blueprint path: blank (this blueprint uses the default setup)
   * if prompted to use existing services, select <InlineUIElement>Create all as new services</InlineUIElement>

1. Click <InlineUIElement>Deploy Blueprint</InlineUIElement>. This may take some time, but remember that free tier deploys shut down after 15 minutes of idle time. Stay close so you can use your new instance before it automatically shuts down.

After setting up the database and FusionAuth instance, Render will create the following variables for your FusionAuth instance:

* `DATABASE_HOST`: the public URL for your database.
* `POSTGRES_PASSWORD`: the database password that FusionAuth uses to connect to your database.
* `POSTGRES_USER`: the database username that FusionAuth uses to connect to your database.
* `FUSIONAUTH_APP_RUNTIME_MODE`: sets your FusionAuth instance to Production mode.

## Set Up FusionAuth

Now you need to configure FusionAuth:

1. In the Render account dashboard, open the <InlineUIElement>Resources</InlineUIElement> section of your blueprint. Click the URL to open the FusionAuth admin UI.

1. Before you can access the FusionAuth admin UI, you must finish the FusionAuth [setup wizard](/docs/get-started/download-and-install/setup) by creating an admin user:
   * Enter an email address, name, and password.
   * Select an option from the <InlineField>How did you hear about us</InlineField> dropdown.
   * Mark the checkbox labelled <InlineUIElement>I accept the terms of the license agreement</InlineUIElement>.
   
   Click <InlineUIElement>Continue setup</InlineUIElement> to create an admin user using these credentials.
1. Congratulations! You now have a working instance of FusionAuth.
