## Overview

FusionAuth can be run on most cloud providers including services like Render. Render is a platform for deploying containerized applications, allowing developers to host web servers, static sites, PostgreSQL, Redis, and cron jobs.

## Prerequisites

* A Render account—the free version is plenty for this tutorial, but will require a paid account to run your application for longer than a month

## Fork the render blueprint

Render allows for projects to be set up manually or via their blueprint mechanism. Blueprints allow for an **Infrastructure as Code** (IaC) setup. This allows you to manage your deployments of multiple services from within one yaml file.

The FusionAuth blueprint gives you a PostgreSQL database and a FusionAuth Web service that both run from a single Render project. While you can run a blueprint directly from our repository, it’s a best practice to run this blueprint from a forked version of the repository in your own GitHub account. Since Render Blueprints stay connected to and are automatically updated from the repository they’re connected to, it’s important to own the repository to keep full control over your own Render instance.

1. Go to [the repository](https://github.com/FusionAuth/fusionauth-render-blueprint)  
2. Click the “Fork” button  
3. Once GitHub finishes, it will give you a URL to your new repository

## Create a new blueprint

From your Render account dashboard, click the “Blueprints” link in the navigation.

Next, click “New Blueprint Instance”.

Render will prompt you for either connecting a repository from a Git provider or pasting a public Git repository into a field. You can proceed either way. If you use the URL, provide the public URL to your repository.

The resulting page will ask you a series of questions for the setup of your project. Give your blueprint the name FusionAuth, specify the branch—in this case main— and you can leave the blueprint path blank, as the blueprint uses the default setup.

If Render detects potential services already existing in your account, it will recommend that you associate the blueprint with them, but for this setup, we’ll use “Create all as new services.” Once these details are set up, click “Deploy Blueprint.”

<Aside type="caution" title="Speeds on free plan">
This process takes some time. While you can run this and walk away, the free tier of Render only allows for 15 minutes of idle time before it shuts a service down. We recommend during this process that you stay close so as not to incur additional slowdown. The free version is also a relatively slow server instance, so note that performance may not be ideal and if speed is important in your test, be sure to try their free trial or a paid account.
</Aside>

<Aside type="note" title="Deployment Details">

The blueprint file sets up the database and FusionAuth instance. It also connects the two via environment variables. The blueprint creates the following variables for your FusionAuth instance:

* DATABASE\_HOST: The public URL for your database  
* POSTGRES\_PASSWORD: The Database password that FusionAuth uses to connect to your database  
* POSTGRES\_USER: The Database username that FusionAuth uses to connect to your database  
* FUSIONAUTH\_APP\_RUNTIME\_MODE: Sets your FusionAuth instance to Production mode

</Aside>

## Running the setup wizard

Once the process is done building out the infrastructure, you need to setup your FusionAuth app with an admin user using our Setup Wizard. 

Navigate to your new FusionAuth web service in your Blueprint’s “Resources” section. 

Render will provide you with a default URL (you can change this later). Click the URL provided.

This page will be your FusionAuth dashboard. Before we can get in, though, we need to finish the FusionAuth setup. To do this, fill out the information for our Setup Wizard. This gives you your administrator user.

## Conclusion and Next Steps

FusionAuth is now completely setup and ready for your needs. From here, go into your FusionAuth dashboard and set up a new application, and you’re ready to attach it to whatever application you’re setting up.

Feel free to explore more [FusionAuth tutorials](https://fusionauth.io/docs/quickstarts) and make sure your environment is tailored to your needs.
