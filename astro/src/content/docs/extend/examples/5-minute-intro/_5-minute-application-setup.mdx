Once you arrive in the FusionAuth admin UI, the first thing you need to do is create an Application. 

If you see a First Time Setup wizard in the main content window, you can click <Breadcrumb>Skip it</Breadcrumb> as you'll be creating things outside of this wizard.

An Application is something that a user can log into. This is the application you are building or that you are migrating to use FusionAuth. Click the <Breadcrumb>Applications</Breadcrumb> menu option on the left side of the page.

![Dashboard to Applications](/img/docs/extend/examples/5-minute-intro/dashboard-applications.png)

This will take you to the listing page for Applications. Next, you'll click the <Breadcrumb>green plus</Breadcrumb> button (the add button) at the top of the page:

![Application Listing](/img/docs/extend/examples/5-minute-intro/application-listing.png)

On the Add Application form, provide a name for your Application (this is only used for display purposes).

Click on the <Breadcrumb>OAuth</Breadcrumb> tab. 

When you click on the <Breadcrumb>OAuth</Breadcrumb> tab, you'll see this form. Start with a simple setup that allows existing users to log into your application. There is no need to add any roles or registration configuration.

![Application Form](/img/docs/extend/examples/5-minute-intro/application-form.png)

Most of the defaults will work, but provide the following items.

* An <InlineField>authorized redirect URL</InlineField>. This is the route/controller in your application's backend that will complete the OAuth grant and obtain a token. For this example, add <InlineFieldValue>http://localhost:3000/oauth-redirect</InlineFieldValue>. You will see example code below for this route. Note that when you type or paste values into this text field you also must click the popup that appears. Otherwise the value will not be saved.
* Specify a valid <InlineField>Logout URL</InlineField>. This is where the user will be redirected to after they are logged out of FusionAuth's OAuth front-end: your application. For this example, set this value to <InlineFieldValue>http://localhost:3000/logout</InlineFieldValue>. You'll see how this is used below.
* Make sure the Authorization Code grant is selected in the <InlineField>Enabled Grants</InlineField>.
* Set the <InlineField>Require registration</InlineField> field to <InlineFieldValue>true</InlineFieldValue>.

Once you have configured the application, you'll click the blue <Breadcrumb>Save</Breadcrumb> button (disk) at the top of the page.

![Save Button Application Form](/img/docs/extend/examples/5-minute-intro/application-form-save.png)

Then, click on the <Breadcrumb>green magnifying glass</Breadcrumb> on the list view next to your application. Scroll down to the <Breadcrumb>OAuth Configuration</Breadcrumb> section and copy the <InlineField>Client Id</InlineField> and <InlineField>Client Secret</InlineField> values off to a text file. You'll use them later.

![Application Details](/img/docs/extend/examples/5-minute-intro/application-details.png)
