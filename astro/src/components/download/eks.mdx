Follow the steps below to install FusionAuth on Kubernetes using Helm. You'll need to have a running kubernetes cluster; for full cluster setup instructions, see our [EKS documentation](https://fusionauth.io/docs/get-started/download-and-install/kubernetes/eks).

You'll need to update `values.yml` with database and Elasticsearch configuration information. For more information, see our [kubernetes installation instructions](https://fusionauth.io/docs/get-started/download-and-install/kubernetes/fusionauth-deployment)

<Steps>

1. Run the following commands:

   ```console
   helm repo add fusionauth https://fusionauth.github.io/charts && \
   curl -o values.yaml https://raw.githubusercontent.com/FusionAuth/charts/master/chart/values.yaml && \
   helm install my-release fusionauth/fusionauth -f values.yaml && \
   export SVC_NAME=$(kubectl get svc --namespace default -l "app.kubernetes.io/name=fusionauth,app.kubernetes.io/instance=my-release" -o jsonpath="&lbrace;.items[0].metadata.name}") && \
   kubectl port-forward svc/$SVC_NAME 9011:9011
   ```

1. Open [http://localhost:9011](http://localhost:9011) in your browser.

</Steps>
