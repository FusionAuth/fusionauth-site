== Overview

A FusionAuth Form Field is an object that can be customized to receive input within a FusionAuth link:/docs/v1/tech/apis/forms[Form].

The following APIs are provided to manage Forms Fields.

* <<Create a Form Field>>
* <<Retrieve a Form Field>>
* <<Update a Form Field>>
* <<Delete a Form Field>>

== Create a Form Field

This API is used to create a new Form Field.

=== Request

[.api-authentication]
link:authentication#api-key-authentication[icon:lock[role=red,type=fas]] Create a Form Field with a randomly generated Id
[.endpoint]
.URI
--
[method]#POST# [uri]#/api/form/field#
--

[.api-authentication]
link:authentication#api-key-authentication[icon:lock[role=red,type=fas]] Create a Form Field with the provided unique Id
[.endpoint]
.URI
--
[method]#POST# [uri]#/api/form/field/`\{fieldId\}`#
--


==== Request Parameters

[.api]
[field]#fieldId# [type]#[UUID]# [optional]#Optional# [default]#defaults to secure random UUID#::
The Id to use for the new Form Field. If not specified a secure random UUID will be generated.

include::docs/v1/tech/apis/_form-field-request-body.adoc[]

=== Response

The response for this API contains the Form Field that was created.

[cols="1,9"]
.Response Codes
|===
|Code |Description

|200
|The request was successful. The response will contain a JSON body.

|400
|The request was invalid and/or malformed. The response will contain an link:errors[Errors] JSON Object with the specific errors.

|401
|You did not supply a valid Authorization header. The header was omitted or your API key was not valid. The response will be empty. See link:authentication[Authentication].

|500
|There was an internal error. A stack trace is provided and logged in the FusionAuth log files. The response will be empty.
|===

== Retrieve a Form Field

This API is used to retrieve a single Form Field by unique Id or all of the configured Form Fields.

=== Request


[.api-authentication]
link:authentication#api-key-authentication[icon:lock[role=red,type=fas]] Retrieve all of the Form Fields
[.endpoint]
.URI
--
[method]#GET# [uri]#/api/form/field#
--

[.api-authentication]
link:authentication#api-key-authentication[icon:lock[role=red,type=fas]] Retrieve a Form Field by Id
[.endpoint]
.URI
--
[method]#GET# [uri]#/api/form/field/`\{fieldId\}`#
--

==== Request Parameters

[.api]
[field]#fieldId# [type]#[UUID]# [required]#Required#::
The unique Id of the Form Field to retrieve.

=== Response

The response for this API contains either a single Form Field or all of the Form Fields. When you call this API with an Id, the response will contain a single Form Field. When you call this API without an Id, the response will contain all of the Form Fields. Both response types are defined below along with an example JSON response.

[cols="1,9"]
.Response Codes
|===
|Code |Description

|200
|The request was successful. The response will contain a JSON body.

|400
|The request was invalid and/or malformed. The response will contain an link:errors[Errors] JSON Object with the specific errors.

|401
|You did not supply a valid Authorization header. The header was omitted or your API key was not valid. The response will be empty. See link:authentication[Authentication].

|404
|The object you requested doesn’t exist. The response will be empty.

|500
|There was an internal error. A stack trace is provided and logged in the FusionAuth log files. The response will be empty.
|===
