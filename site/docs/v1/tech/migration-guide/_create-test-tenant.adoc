
It is best to create a separate tenant for migration testing. Tenants logically isolate configuration settings and users. If a migration goes awry or you need to redo it after tweaking the script, you can delete the test tenant and start with a clean system. To add a tenant, navigate to [breadcrumb]#Tenants# and choose the green plus sign.

image::migration-guide/{migration_source_dir}/list-of-tenants-add-highlighted.png[Adding a tenant.,width=1200,role=bottom-cropped]

Give it a descriptive name like `{migration_source_name} import test`. You shouldn't need to modify any of the other configuration options to test importing users, but feel free to change any that you want to.

Save the tenant.

image::migration-guide/{migration_source_dir}/add-tenant.png[The tenant creation screen.,width=1200,role={add-tenant-image-role}]

Record the Id of the tenant, which will be something like `25c9d123-8a79-4edd-9f76-8dd9c806b0f3`. You'll use this later.

image::migration-guide/{migration_source_dir}/list-of-tenants.png[The tenant list.,width=1200,role=bottom-cropped]
